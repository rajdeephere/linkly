# ADR-0005 — Default 302 (temporary) redirect, not 301

**Status:** Accepted (Day 2) · **Date:** 2026-07-15

## Context
The resolver must choose an HTTP redirect status. A 301 (permanent) is aggressively cached by browsers
and proxies; a 302 (found/temporary) is not.

## Decision
Resolve returns **302** by default.

## Consequences
- **Analytics keep flowing** — a 301 is cached forever client-side, so future clicks never reach us and
  click tracking silently stops. 302 routes every click through the resolver.
- **Edits propagate** — if the destination is changed later, 301-cached browsers keep hitting the old
  URL forever; 302 always re-resolves.
- Marginally less "SEO link juice" is passed vs a 301, and slightly more requests hit the edge (cheap).

## Alternatives
- **301 everywhere**: better for static SEO links, fatal for a *managed*, editable, analytics-bearing
  link — which is the whole product.

## Revisit if
A user needs a permanent, SEO-weighted link → offer **301 as an opt-in per link**, accepting that
analytics and editability degrade for that link.
